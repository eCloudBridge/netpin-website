Quick Start

Get up and running with Netpin in minutes:

Clone the Repository
git clone https://github.com/netpin/kubeapps.git
cd kubeapps
Start the Platform
docker compose up -d

This starts all services: API Gateway, Auth, Cluster Manager, IDI Service, Kubernetes API, and Frontend.

Access the Dashboard

Open your browser and navigate to http://localhost:3000

Create an account or log in with the default credentials.

Connect Your First Cluster

Go to Clusters ‚Üí Add Cluster and either:

  • Connect a cloud provider (AWS, GCP, Azure) to auto-discover clusters
  • Manually add a cluster with kubeconfig

Platform Architecture

Netpin is built as a microservices architecture with the following core components:

Service Port Description
api-gateway 3000 Main entry point, routes requests to backend services
auth-service 8001 JWT authentication, user management, RBAC
cluster-manager 8002 Cloud provider integration, cluster discovery
kubernetes-api 8000 Kubernetes operations, Helm releases, topology
idi-service 8003 Infrastructure Debt Index calculation and forecasting
frontend 5173 React-based web UI with Vite

Key Concepts

Infrastructure Debt Index (IDI)

The IDI is a comprehensive score (0-100) that quantifies the technical debt in your infrastructure. A lower score indicates a healthier infrastructure. The IDI considers:

  • Security findings ‚Äî Vulnerabilities, misconfigurations, exposed secrets
  • Compliance gaps ‚Äî Policy violations, best practice deviations
  • Resource efficiency ‚Äî Over-provisioning, unused resources
  • Operational risks ‚Äî Single points of failure, missing redundancy

Application Stacks

Stacks represent logical groupings of Kubernetes resources that make up an application. Netpin can auto-detect stacks or you can define them manually.

Deployment Gating

Integrate Netpin with your CI/CD pipeline to automatically block deployments when:

  • IDI score exceeds a threshold
  • Critical security findings are detected
  • Dependencies have known vulnerabilities
