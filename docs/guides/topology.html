Topology Visualization

Interactive visualization of your Kubernetes cluster resources, their relationships, and dependencies.

Overview

The Topology view provides a real-time visual map of all resources in your Kubernetes cluster. Resources are displayed as nodes connected by edges representing their relationships.

Resource Types

The topology visualizes these Kubernetes resources:

Resource Color Description
Namespace Purple border Groups resources within the same namespace
Deployment Blue Application workloads with replicas
StatefulSet Cyan Stateful application workloads
DaemonSet Amber Node-level workloads
Service Green Network endpoints for workloads
Ingress Indigo External traffic routing
ConfigMap Gray Configuration data
Secret Red Sensitive configuration
PVC Pink Persistent storage claims

Interactions

Navigation

  • Pan: Click and drag the background
  • Zoom: Scroll wheel or pinch gesture
  • Fit: Double-click background to fit all nodes

Node Actions

  • Click: Select node and view details panel
  • Double-click: Expand/collapse connected nodes
  • Drag: Reposition node

Details Panel

When a node is selected, the side panel shows:

  • Resource name and type
  • Namespace
  • Labels and annotations
  • Status and health
  • Related findings (if any)
  • YAML manifest preview

Filters

Filter the topology view using the control panel:

By Namespace

Select specific namespaces to display or hide system namespaces like kube-system.

By Resource Type

Toggle visibility of specific resource types (Deployments, Services, etc.).

By Status

Filter by health status: Healthy, Warning, Error.
