Quick Start Guide

Get Netpin up and running in under 5 minutes. This guide walks you through the essential steps to start monitoring your Kubernetes infrastructure.

Prerequisites: Docker and Docker Compose installed on your system.

Step 1: Clone and Start

Clone the Netpin repository and start all services with Docker Compose:

# Clone the repository
git clone https://github.com/netpin/kubeapps.git
cd kubeapps

# Start all services
docker compose up -d

# Check services are running
docker compose ps

You should see all services running:

NAME                     STATUS
netpin-postgres          healthy
netpin-redis             healthy
netpin-auth              running
netpin-cluster-manager   running
netpin-k8s-api           running
netpin-idi-service       running
netpin-api-gateway       running
netpin-frontend          running

Step 2: Access the Dashboard

Open your browser and navigate to:

http://localhost:3000

Create Your Account

Click "Sign Up" to create a new account:

  • Enter your email address
  • Choose a strong password (min 8 characters)
  • Enter your full name
  • Click "Create Account"

Step 3: Create a Project

Projects organize your clusters and resources. Create your first project:

Navigate to Dashboard

After login, you'll land on the main Dashboard.

Click "+ New Project"

In the project selector dropdown, click "Create New Project".

Enter Project Details

Give your project a name (e.g., "Production", "Staging") and optional description.

Step 4: Connect a Cloud Provider

Netpin can automatically discover your Kubernetes clusters from cloud providers.

AWS (EKS)

# In the Netpin UI:
1. Go to Clusters ‚Üí Cloud Providers
2. Click "Add Provider"
3. Select "AWS"
4. Enter your AWS credentials:
   - Access Key ID
   - Secret Access Key
   - Region (e.g., ap-south-1)
5. Click "Connect"

# Netpin will automatically discover all EKS clusters in that region

IAM Permissions Required: Your AWS credentials need the following permissions:
  • eks:ListClusters
  • eks:DescribeCluster
  • Kubernetes RBAC access to the cluster

Manual Cluster (Kubeconfig)

For clusters not in a supported cloud provider, add them manually:

# In the Netpin UI:
1. Go to Clusters ‚Üí Add Cluster
2. Select "Manual (Kubeconfig)"
3. Paste your kubeconfig content
4. Click "Add Cluster"

Step 5: Deploy the Netpin Agent

The Netpin Agent runs inside your cluster and collects metrics, findings, and inventory data.

# Apply the agent deployment
kubectl apply -f https://raw.githubusercontent.com/netpin/kubeapps/main/agent/deploy/agent.yaml

# Or from the local repo
kubectl apply -f agent/deploy/agent.yaml

Configure the agent with your Netpin server details:

apiVersion: v1
kind: ConfigMap
metadata:
  name: netpin-agent-config
  namespace: netpin-system
data:
  NETPIN_SERVER_URL: "http://your-netpin-server:3000"
  NETPIN_API_KEY: "your-api-key"
  CLUSTER_ID: "your-cluster-id"
