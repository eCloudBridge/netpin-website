Application Stacks

Application Stacks represent logical groupings of Kubernetes resources that work together to deliver a business capability.

What are Stacks?

A Stack is a collection of related Kubernetes resources (Deployments, Services, ConfigMaps, etc.) that form a cohesive application or microservice. Stacks allow you to:

  • Track IDI scores at the application level
  • View findings grouped by application
  • Understand dependencies between applications
  • Gate deployments based on stack health

Auto-Detection

Netpin can automatically detect stacks based on common labeling patterns: 

# Resources with these labels are grouped into stacks
app.kubernetes.io/name: my-app
app.kubernetes.io/part-of: my-stack
app: my-app

Manual Stack Definition

You can also manually define stacks by specifying a label selector: 

{
  "name": "Payment Service",
  "selector": {
    "app": "payment",
    "tier": "backend"
  },
  "namespaces": ["production", "staging"]
}

Stack Dependencies

Netpin tracks dependencies between stacks based on Service references, helping you understand the impact of changes.

Metric Description
Fan-out Number of other stacks this stack depends on
Fan-in Number of stacks that depend on this stack
Criticality Calculated importance based on dependencies
