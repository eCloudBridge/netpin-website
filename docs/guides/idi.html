Infrastructure Debt Index (IDI)

The Infrastructure Debt Index is Netpin's proprietary scoring system that quantifies technical debt across your infrastructure, helping you make data-driven decisions about maintenance, upgrades, and deployments.

What is Infrastructure Debt?

Infrastructure debt is the accumulated cost of deferred maintenance, outdated configurations, security vulnerabilities, and operational inefficiencies in your infrastructure. Like financial debt, it accrues "interest" over time ‚Äî small issues become larger problems that are more expensive to fix.

Scoring Algorithm

The IDI score is calculated using weighted categories:

IDI = (Security √ó 0.40) + (Compliance √ó 0.25) + (Efficiency √ó 0.20) + (Reliability √ó 0.15)

Where each category score (0-100) is calculated from:
- Number of findings
- Severity of findings (Critical: 10, High: 5, Medium: 3, Low: 1)
- Age of findings (older issues weight more heavily)

Category Weights

Category Weight Rationale
Security 40% Security vulnerabilities pose the highest risk to the organization
Compliance 25% Policy adherence ensures consistency and auditability
Efficiency 20% Resource optimization impacts costs and performance
Reliability 15% Reliability issues affect uptime and user experience

Finding Categories

Security Findings

  • Container images with known CVEs
  • Pods running as root
  • Missing network policies
  • Exposed secrets in environment variables
  • Privileged containers
  • Missing pod security policies
  • Overly permissive ServiceAccount permissions

Compliance Findings

  • Missing required labels (app, version, owner)
  • Resources without limits defined
  • Deprecated API versions in use
  • Non-compliant naming conventions
  • Missing annotations for tooling integration

Efficiency Findings

  • Over-provisioned CPU/memory requests
  • Unused ConfigMaps and Secrets
  • Orphaned PersistentVolumeClaims
  • Idle deployments (no traffic)
  • Duplicate container images

Reliability Findings

  • Single replica deployments
  • Missing liveness/readiness probes
  • No PodDisruptionBudgets
  • Missing anti-affinity rules
  • No horizontal pod autoscaling

IDI Forecasting

Netpin uses historical data and trend analysis to forecast your IDI score:

Forecast Periods

  • 7-day forecast: Short-term trend with high confidence
  • 14-day forecast: Medium-term projection
  • 30-day forecast: Long-term trend if no action taken

Key Metrics

  • Days to Critical: Estimated days until IDI reaches critical threshold (85)
  • Days to High: Estimated days until IDI reaches high threshold (70)
  • Trend Direction: Improving, stable, or deteriorating
  • Average Daily Change: How much the IDI changes per day on average

CI/CD Integration Example

# GitHub Actions example
- name: Check Deployment Gate
  run: |
    RESPONSE=$(curl -s -X POST \
      -H "Authorization: Bearer $NETPIN_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"project_id": "$PROJECT_ID", "stack_id": "$STACK_ID"}' \
      https://netpin.example.com/api/v2/deploy/gate)
    
    DECISION=$(echo $RESPONSE | jq -r '.decision')
    
    if [ "$DECISION" = "blocked" ]; then
      echo "Deployment blocked due to high IDI score"
      exit 1
    fi
