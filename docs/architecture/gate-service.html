Gate Service

The Gate Service acts as a quality assurance firewall for your deployments, blocking risky changes based on IDI scores and policy definitions.

Key Features

  • Policy Enforcement: Block deployments if IDI score > Threshold.
  • CI/CD Integration: Webhooks for Jenkins, GitHub Actions, and GitLab CI.
  • Vulnerability Gating: Prevent deployment of images with Critical CVEs.
  • Approval Workflow: Manual override capabilities for emergency fixes.

Deployment Flow

How the Gate Service interacts with your CI/CD pipeline:

sequenceDiagram
    participant CI as CI/CD Pipeline
    participant Gate as Gate Service
    participant IDI as IDI Service
    participant K8s as Kubernetes Cluster

    CI->>Gate: POST /check-deployment (Manifest)
    activate Gate
    Gate->>IDI: Calculate Projected Score
    IDI-->>Gate: IDI Score: 85 (High Risk)
    
    alt Score < Threshold
        Gate-->>CI: 200 OK (Allowed)
        CI->>K8s: Apply Manifest
    else Score > Threshold
        Gate-->>CI: 403 Forbidden (Blocked)
        Note right of CI: Deployment Failed.\nRisk too high.
    end
    deactivate Gate
